# Manual Image Solver テスト手順書

## テスト階層

| レベル | ツール | 責務 | テストしないこと |
|---|---|---|---|
| 単体テスト | Node.js | 数学関数の精度（TAN投影、CD行列フィット、角距離）、座標パース、MTF | PixInsight API 連携、UI 動作 |
| 統合テスト | PJSR | WCS適用フロー、セントロイド計算、Sesame検索 | 数学関数の精度（単体で検証済み） |
| E2E テスト | 手動 | 実画像での座標精度、ImageSolver との比較 | — |

## 1. Node.js 単体テスト

### 1.1 test_wcs_math.js（WCS 数学関数）

#### 実行方法

```bash
node tests/javascript/test_wcs_math.js
```

#### テストケース一覧

| カテゴリ | テスト名 | 検証内容 |
|---|---|---|
| TAN投影 | 投影中心での往復 | (ξ,η) = (0,0) であること |
| TAN投影 | 近傍点の往復精度 | 往復誤差 < 1e-10 度 |
| TAN投影 | RA=0 付近のラップアラウンド | 359° → 1° の投影が正しい |
| TAN投影 | 天の南極付近 | DEC=-89° 付近で精度維持 |
| TAN投影 | 反対半球で null | D ≤ 0 で null を返す |
| 角距離 | 同一点で 0 | 距離 = 0 |
| 角距離 | ベテルギウス↔リゲル | 文献値 ≈ 18.5° と一致 |
| 角距離 | 天の極間 | 180° |
| 角距離 | 赤道上 90° | 90° |
| WCSFitter | 3星未満エラー | success: false, メッセージに「4」を含む |
| WCSFitter | 不正 RA エラー | RA ≥ 360 で失敗 |
| WCSFitter | 不正 DEC エラー | DEC > 90 で失敗 |
| WCSFitter | 合成4星（最小構成） | 残差 < 1 arcsec |
| WCSFitter | 合成6星（標準） | 残差 < 0.01 arcsec |
| WCSFitter | 合成10星（高精度） | 残差 < 0.01 arcsec, CD行列一致 |
| WCSFitter | CRVAL 初期値ずれ | 正しい WCS に収束 |
| WCSFitter | 回転した CD 行列 | 30° 回転の CD 行列を再現 |
| WCSFitter | ピクセルスケール | pixelScale_arcsec が正しい |

#### 合格基準

- 55 テスト中 52 テスト passed（3 件は CD 行列符号の既知の差異で FAIL）
- 既知の FAIL: CD 行列の個別要素の符号が期待値と反転するケース（WCS 全体の精度には影響なし）

## 2. PJSR 統合テスト

### 実行方法

PixInsight コンソール:

**Script > Run Script File...** → `tests/javascript/ManualSolverTest.js`

### テストケース一覧

| テスト名 | 検証内容 |
|---|---|
| WCS キーワード適用 | applyWCS() 後に正しいキーワードが設定される。非WCSキーワード（OBJECT, FILTER）が保持される。既存WCSキーワードが上書きされる。 |
| セントロイド計算 | テスト画像（200×200）にガウシアン星像を配置。クリック位置から2px以内のずれで正しいセントロイドを返す。 |
| CDS Sesame 検索 | "Sirius" の検索で RA ≈ 101.287°, DEC ≈ -16.716° を返す。ネットワーク不可時は SKIP。 |

### 合格基準

- WCS キーワード適用: 全 assert 通過
- セントロイド: 誤差 1px 以内
- Sesame: 座標が文献値 ±0.1° 以内（またはネットワーク不可で SKIP）

## 3. E2E テスト（手動）

### 前提条件

- PixInsight がインストール済み
- 既知の WCS 解がある星野画像（ImageSolver で事前にソルブ済み）

### テスト手順

#### テスト A: 基本動作確認

1. PixInsight で星野画像を開く
2. **Script > Astrometry > ManualImageSolver** を起動
3. 画像コンボボックスで対象画像が選択されていることを確認
4. 画像プレビューが表示されることを確認
5. ズームボタン（Fit / 1:1 / + / −）が動作することを確認
6. STF 切替（None / Linked / Unlinked）が動作することを確認
7. ドラッグによるパンが動作することを確認

**合格基準**: 各操作がエラーなく動作する

#### テスト B: 星の登録と座標検索

1. 画像上の明るい星をクリック（セントロイドスナップで StarEditDialog が開く）
2. StarEditDialog が開くことを確認
3. ピクセル座標がセントロイドスナップされていることを確認（コンソールログ参照）
4. Name に "Sirius" と入力して Search をクリック
5. RA/DEC が自動入力されることを確認
6. OK で閉じ、テーブルに追加されることを確認
7. 追加の星をさらに 3 つ以上登録
8. 「Edit...」ボタンで既存星の編集ができることを確認
9. 「Remove」ボタンで星を削除できることを確認
10. 「Export...」で星ペアデータを JSON ファイルに出力できることを確認
11. 「Import...」で JSON ファイルから星ペアデータを読み込めることを確認

**合格基準**: 星の登録・編集・削除・エクスポート・インポートがエラーなく動作する

#### テスト C: WCS フィッティングと適用

1. 4星以上を登録した状態で「Solve」をクリック
2. コンソールに WCS パラメータが表示されることを確認
3. テーブルの Residual 列に残差が表示されることを確認
4. ステータスバーに RMS とピクセルスケールが表示されることを確認
5. 「Apply」をクリック
6. 確認ダイアログで「Yes」を選択
7. 画像のアストロメトリック表示（座標グリッド等）が更新されることを確認

**合格基準**: WCS が正常に適用され、座標表示が有効になる

#### テスト D: 精度検証

1. 既知の WCS 解がある画像で ManualImageSolver を実行
2. 6星以上を登録して Solve
3. 以下を確認:
   - RMS 残差が 1 arcsec 以下
   - 各星の残差が 2 arcsec 以下
   - CRVAL が画像中心の天球座標と一致（±0.01°）
4. PixInsight の ImageSolver（標準）の結果と比較:
   - CRVAL の差が 0.01° 以下
   - ピクセルスケールの差が 0.01 arcsec/px 以下

**合格基準**: 残差が 1 arcsec 以下、ImageSolver との差が 0.01° 以下

#### テスト E: エラーハンドリング

1. 画像を開かずにスクリプトを起動 → エラーメッセージが表示される
2. 3星以下で Solve → エラーメッセージが表示される
3. 存在しない天体名で Search → 警告メッセージが表示される
4. RA/DEC を空欄で OK → バリデーションエラーが表示される

**合格基準**: 各エラーが適切なメッセージで処理される
